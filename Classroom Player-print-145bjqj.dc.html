Welcome! This lesson shows how FiveM scripts work, why Lua is used, and how client ↔ server events communicate. Follow the steps below and grab the downloads in the Files tab.

What you'll learn
  • Client vs server scopes — and where each print() shows up
  • Create a minimal resource with fxmanifest.lua
  • Register your first command with RegisterCommand
  • Send events client→server and server→client
  • Handle async with Citizen.Wait to avoid nil surprises

Build it: a minimal resource

fxmanifest.lua
Copy 
fx_version 'cerulean'
game 'gta5'

author 'Quasar University'
description 'Zero to Hero — Lesson 1'

client_script 'client.lua'
server_script 'server.lua'
client.lua
RegisterCommand('hello', function()
  TriggerServerEvent('zh:hello')
end)

-- server replies to us
RegisterNetEvent('zh:reply', function(msg)
  print(msg)
end)
server.lua
RegisterNetEvent('zh:hello', function()
  local src = source
  TriggerClientEvent('zh:reply',
    src, 'Hello from server!')
end)
Tip: always declare local — global vars add a small delay on calls. Test in-game with /hello and watch the server console.
